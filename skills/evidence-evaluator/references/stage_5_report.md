# Stage 5: Evidence Evaluation Report Synthesis

## Purpose (v2)
Produce the **Evidence Evaluation Report** — the primary output of the pipeline. Four parts:
1. **Structured Findings Summary** (deterministic, no LLM) — assembles all stage outputs
2. **Narrative Summary** (LLM call) — 500–800 words for clinician audience, findings only
3. **Optional Suggested Score** (rule engine) — 1–5 heuristic, explicitly labeled as pending calibration
4. **Optional Markdown Export** — full report rendered as a `.md` file for sharing/archiving

---

## Part 2: Narrative Summary (LLM)

**Length:** 500–800 words.
**Audience:** Clinicians — use clinical language, not academic jargon.
**Tone:** Objective, informative, findings-focused.
**Critical rule:** Do NOT render a verdict. Surface what was found and let the clinician judge.

**Structure the narrative around:**
1. Study design and population (1–2 sentences)
2. Statistical robustness findings — FI, LTFU rule, power (2–3 sentences)
3. Clinical benchmarking — effect vs. MCID, NNT interpretation (2–3 sentences)
4. Bias risk summary — most important domain findings (2–3 sentences)
5. Notable strengths and limitations (2–3 sentences)
6. Closing — what the clinician should weigh when interpreting these findings

**What NOT to include:**
- No final verdict ("This is good evidence" / "This paper should not be trusted")
- No score or numeric summary (that's the optional module)
- No extrapolation beyond the paper's stated scope

## Part 3: De-duplication Rules (apply before computing optional score)

These rules prevent double-counting the same underlying limitation:

| Rule | Detail |
|---|---|
| **Statistical stability dimension** | Among {power < 0.80, N < domain standard, NNT > threshold} → apply **only the most severe** single deduction |
| **Diagnostic case-control overlap** | Stage 2 case-control deduction + QUADAS-2 patient selection = same bias → apply once |
| **QUADAS-2 cap** | Raw domain deductions capped at −2 total |
| **GRADE upgrade cap** | Capped at +1 total regardless of factors met |
| **LTFU-FI hard rule** | −2, highest priority, **never deduplicated** |

### Rendering Rules

- **Emoji status indicators** use a consistent scheme: 🟢 favorable / 🟡 borderline / 🔴 concern. Use these in all table cells — they make the report scannable at a glance.
- **Conditional blocks** (human review banner, Phase 0/I disclaimer, Tier 4 warning, LTFU hard rule row) are only rendered when the corresponding flag is true. Do not render empty or n/a banners.
- **Skipped stages** (Stage 2 + 3 for phase_0_1; Stage 3 for diagnostic) render their section headers with a single line: `*This stage was not run for [study_type] studies.*`
- **Retracted/excluded papers** suppress all sections except the exclusion block:

```markdown
# Evidence Evaluation Report

> ⛔ **PAPER EXCLUDED — DATA INTEGRITY CONCERN**
> This study has been identified as retracted or has confirmed data integrity issues.
> **Reason:** [retraction | undisclosed conflict of interest]
> **Source:** [Retraction Watch / institutional notice]
> No evidence evaluation is provided.
```

### Output filename convention

```
evidence_report_[first_author_lastname]_[year]_[PMID_or_DOI_suffix].md
```

Example: `evidence_report_smith_2024_pmid38291045.md`

If PMID and DOI are both unavailable: `evidence_report_[first_author_lastname]_[year]_[job_id_short].md`

## Handling Special Cases in Stage 5

### Retracted / Excluded Paper
Suppress all sections. Render exclusion block only (see Markdown template above).

### Phase 0/I Study
Add required disclaimer at top of report:
> *"This is a Phase 0/I safety trial. The report reflects study methodology quality only and does not represent efficacy evidence strength. Stages 2 and 3 were not run. Score range is locked at 1–2."*

### Human Review Flag
Add banner at top:
> *"⚠ One or more extraction fields had low confidence (< 0.70). Flagged fields: [list]. Results are provisional and should be verified against the source paper."*
